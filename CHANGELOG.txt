Changes from v31.1.0: Exhaustive self-audit — 5 code fixes, 4 new
  audit infra lessons. 0 new checks (stays at 285).

  F1: [MED] Scope shadow detector keyword list missing 'function '.
      Nested function definitions (function inner inside function outer)
      did not increment depth. Inner function's 'end' would decrement
      outer depth, causing premature function exit and corrupted scope
      tracking for all subsequent variables in the outer function.
      Fixed: added 'function ' to keyword list ['if ','for ',...,'begin',
      'function '], matching block_re used by all other depth-tracking
      extractors.
  F2: [MED] _py_extract function name regex r'[ \-]' required a space
      or hyphen after the function name. Bare declarations (function foo
      with no --description or --argument-names) have only a newline
      after the name, causing _py_extract to silently return empty output.
      Fixed: changed to r'([ \-]|$)' to match end-of-line.
  F3: [MED] PASS 2 STEP 2 ANALYSIS bash block used $S (script path)
      but never defined it. S was defined in the STEP 1 bash block, but
      each ```bash block is a separate shell invocation — variables
      don't carry across. VERSION CONTENT check would always MISMATCH.
      Fixed: added S=/home/claude/audit-src/script.fish to STEP 2 block.
  F4: [LOW] EXTRACTORS audit infra lesson listed wrong extractors:
      included "comment coverage" (no depth tracking) and missed
      "inline comment density" and "completions flags" (both use
      depth tracking). Count was 6, actual is 7.
      Fixed: corrected list and count to 7.
  F5: [LOW] BARE COREUTILS set filter used \b word boundary on cmd
      name: re.search(r'\bset\b.*\b' + cmd + r'\b', line). Since
      underscore is a word character, \brm\b never matches 'rm' in
      'rm_path', so 'set -l rm_path /tmp' was not filtered — producing
      a false positive (reported as bare coreutil when it's a var name).
      Fixed: removed word boundary on cmd, using substring match.
