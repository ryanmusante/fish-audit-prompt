 AUDIT INFRA LESSONS

    STEP1  SIZE GATE truncation: low line limits silently lose sections
           from large scripts. Skeleton extraction preferred over head -N.
    GROUP_D  Scope shadow detector escape layers: bash double-quote →
           python raw string → regex. Each layer halves backslashes.
           Test with actual Python before deploying multi-layer escapes.
           Always grep for full backslash run count across the entire
           block after fixing escapes.
    EXTRACTORS  All depth-tracking Python extractors (7 total) assume

           check 215 (fish_indent) before trusting extractor output.
    PASS3  GREP-C DOUBLE OUTPUT: `grep -c` on 0-match returns "0"
           (exit 1) → fallback also outputs "0" → two values captured.
           Fix: Fish `set c $c[-1]`; bash `| tail -1`. Applies to ALL
           rg → grep → echo fallback chains producing counts.

STEP 2: FALLBACKS (bash-only — do not fish-ify)

  Requires /bin/bash (uses arrays, ${args[-1]}, ${range%%:*}).

  ```bash
  command -v rg  >/dev/null 2>&1 || rg()  { grep -P --color=never "$@"; }
  command -v bat >/dev/null 2>&1 || bat() {
    local args=() range=""
    while [[ $# -gt 0 ]]; do
      case "$1" in --plain) shift ;; -r) shift; range="$1"; shift ;; *) args+=("$1"); shift ;; esac
    done
    if [[ -n "$range" ]]; then sed -n "${range%%:*},${range##*:}p" "${args[-1]}"; else cat "${args[@]}"; fi
  }
  command -v fd  >/dev/null 2>&1 || fd() { local pat="$1"; shift; find "$@" -name "$pat" 2>/dev/null; }
  export -f rg bat fd 2>/dev/null; true

  # ── Hardening defaults (applied after STEP 2, before data collection) ──
  ```

EVIDENCE RULE:

  If no supporting evidence exists in the collected data, mark as UNVERIFIED.
  Never infer findings without data. Prefer 0 findings to 1 fabricated finding.
  Findings lacking evidence will be rejected during PASS 3 collation.

TOOL MANDATES:
  RUST FIRST    rg over grep, fd over find, bat over cat/less.
                Fall back only if unavailable.
  PARALLEL      Every independent rg/fd call backgrounded with & + wait.
                Group related checks into ( ... ) & subshells. No serial
                rg loops.
  BATCH         Compound commands in single tool calls. Multiple rg in
                one subshell. One bash_tool call per pass, not per check.
  PYTHON3       Primary for function body extraction and line iteration.
                NOTE: All Python extractors assume fish_indent-formatted input.
                Single-line `begin; cmd; end` constructs cause incorrect depth
                tracking. Ensure check 215 (fish_indent) passes before relying
                on extractor output for depth-sensitive checks.
                Define _py_extract helper (see STEP 1) for depth-tracking.
                fish -c extractors fail on some fish versions due to $
                escaping in regex. Use python3 for: function lengths,
                function body extraction, scope shadow detection, tmpfile
                lifecycle, comment coverage, numeric var init, in-function
                globals, command prefix counts, bare coreutils detection.
  PCRE2 GUARD   rg -oP/-cP requires PCRE2. If unavailable, fall back
                to rg -o + grep -oP, or restructure the pattern.
                rg -Un (multiline) needs rg ≥11; fall back to awk for
                adjacent-line patterns if rg multiline unavailable.
  OUTPUT CAPS   Every rg command producing multi-line output MUST pipe
                through | head -N. Use: head -30 for high-hit patterns
                (set, sudo, command), head -50 for medium, head -20 for
                low-hit. Exceptions: rg -c (count-only) needs no cap.

  ```bash
        p13)    phases="P13 (gap analysis)" ;;
        p14)    phases="P14 (v3.1.0-v3.46.0 specifics)" ;;
        *)      phases="unknown ($fname)" ;;
      esac
      echo "WARNING: $f has $lines lines, truncating to 500" >&2
      echo "[INFO] SIZE GATE: $f ($phases) truncated from $lines to 500 lines — affected phase checks may have incomplete data" >> "$O"
      # ── Recovery manifest: list lost sections for targeted re-collection ──
      grep '^=== .* ===$' "$f" > "$D/${fname}-full-sections.txt"
      # Skeleton extraction: keep all section headers + first 5 data lines per section
      awk '/^=== .* ===$/{header=1;count=0;print;next} header{count++;if(count<=5)print}' "$f" > "$f.tmp" && mv "$f.tmp" "$f"
      grep '^=== .* ===$' "$f" > "$D/${fname}-kept-sections.txt"
      comm -23 <(sort "$D/${fname}-full-sections.txt") <(sort "$D/${fname}-kept-sections.txt") > "$D/${fname}-lost.txt"
      lost_count=$(wc -l < "$D/${fname}-lost.txt")
      if [ "$lost_count" -gt 0 ]; then
        echo "[INFO] SIZE GATE RECOVERY: $lost_count sections lost in $fname — manifest at $D/${fname}-lost.txt" >> "$O"
      fi
    fi
  done

  # ── EMPTY SECTION GUARD: detect sections with 0 data lines ──
  for f in "$D"/p*.txt; do
    while IFS= read -r header; do
      section=$(echo "$header" | sed 's/^=== //;s/ ===$//')
      # Get line count between this header and the next (or EOF)
      linenum=$(grep -nF "$header" "$f" | head -n 1 | cut -d: -f1)
      nextnum=$(awk -v start="$linenum" 'NR>start && /^=== .* ===$/{print NR; exit}' "$f")
      : "${nextnum:=$(wc -l < "$f")}"
      content_lines=$((nextnum - linenum - 1))
      if [ "$content_lines" -le 0 ]; then
        echo "[INFO] EMPTY SECTION: $section in $(basename "$f") — rg returned 0 results or section truncated by SIZE GATE" >> "$O"
      fi
    done < <(grep '^=== .* ===$' "$f")
  done

  echo "=== DATA COLLECTION COMPLETE ==="
  wc -l "$D"/*.txt
  ```
